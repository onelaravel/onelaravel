Kiến trúc tiêu chuẩn cho từng module trong One Laravel

Cấu trúc module thực tế:

ModuleName/
├── Models/                    # Eloquent Models
│   └── ModuleName.php
├── Repositories/              # Data Access Layer
│   ├── ModuleNameRepository.php
│   └── ModuleNameRepositoryInterface.php
├── Services/                  # Business Logic
│   ├── ModuleNameService.php
│   └── ModuleNameServiceInterface.php
├── Http/                      # HTTP Layer
│   ├── Controllers/          # Controllers (không phân Api/Web/Admin)
│   │   └── ModuleNameController.php
│   ├── Requests/             # Form Requests
│   │   └── StoreModuleNameRequest.php
│   └── Resources/            # API Resources
│       └── ModuleNameResource.php
├── Masks/                     # Data Masks (Steak Masks)
│   └── ModuleNameMask.php
├── Providers/                 # Route Service Providers
│   └── ModuleNameRouteServiceProvider.php
└── ModuleServiceProvider.php  # Hoặc BootstrapProvider.php
    # Đăng ký bindings và RouteServiceProvider

Ví dụ thực tế (User Module):
├── Models/
│   └── User.php
├── Repositories/
│   ├── UserRepository.php
│   └── UserRepositoryInterface.php
├── Services/
│   ├── UserService.php
│   └── UserServiceInterface.php
├── Http/
│   ├── Controllers/
│   │   └── UserController.php
│   ├── Requests/
│   │   ├── StoreUserRequest.php
│   │   └── UpdateUserRequest.php
│   └── Resources/
│       └── UserResource.php
├── Masks/
│   └── UserMask.php
├── Providers/
│   └── UserRouteServiceProvider.php
└── ModuleServiceProvider.php

Lưu ý:
- Controllers KHÔNG phân Api/Web/Admin trong module
- Routes được đăng ký trong RouteServiceProvider sử dụng Core\System
- ModuleServiceProvider đăng ký bindings (Repository, Service)
- Routes được đăng ký qua System::context()->module()->controller()->group()
